# Lattice-Based Cryptography: How ML-KEM and ML-DSA Work

## What Is a Lattice?

A lattice is a regular grid of points in n-dimensional space, generated by integer combinations of a set of basis vectors. In two dimensions, think of a grid of dots on graph paper, but rotated and stretched.

Formally, given n linearly independent vectors b1, b2, ..., bn in R^n, the lattice L is:

    L = { a1*b1 + a2*b2 + ... + an*bn : a1, a2, ..., an are integers }

The security of lattice-based cryptography relies on the fact that certain problems on high-dimensional lattices are computationally hard — even for quantum computers.

## The Hard Problems

### Shortest Vector Problem (SVP)

Given a lattice basis, find the shortest non-zero vector in the lattice. In low dimensions (2-3), this is easy to visualize and solve. In hundreds of dimensions, the best known algorithms (both classical and quantum) take exponential time.

### Learning With Errors (LWE)

Given a system of approximate linear equations over a finite field:

    b = A * s + e (mod q)

where A is a known matrix, s is a secret vector, and e is a small error vector, recover s.

Without the error term, this is just solving a linear system (easy). The small error makes it hard — and this hardness is what ML-KEM and ML-DSA rely on.

### Module-LWE

ML-KEM and ML-DSA use a structured variant called Module-LWE, where the matrix A has a special algebraic structure (it lives in a module over a polynomial ring). This structure makes the algorithms faster and the keys smaller, while preserving the hardness guarantee.

## How ML-KEM Works (Simplified)

ML-KEM is a Key Encapsulation Mechanism (KEM). Its job is to let two parties establish a shared secret over an insecure channel.

### Key Generation

1. Sample a random matrix A from a public seed.
2. Sample a secret vector s and an error vector e (both small).
3. Compute t = A * s + e (mod q).
4. The public key is (A, t). The secret key is s.

### Encapsulation (Sender)

1. Sample a random vector r and error vectors e1, e2 (all small).
2. Compute u = A^T * r + e1 (mod q).
3. Compute v = t^T * r + e2 + encode(message) (mod q).
4. The ciphertext is (u, v). The shared secret is derived from the message.

### Decapsulation (Receiver)

1. Compute v - s^T * u = message + (small noise) (mod q).
2. Round to recover the original message.
3. Derive the shared secret from the message.

The small noise cancels out because s^T * (A^T * r + e1) is close to (A * s)^T * r = t^T * r - e^T * r, and the error terms are small enough to round away.

## How ML-DSA Works (Simplified)

ML-DSA is a digital signature scheme. Its job is to let a signer prove that a message is authentic.

### Key Generation

1. Sample a random matrix A.
2. Sample secret vectors s1 and s2 (both small).
3. Compute t = A * s1 + s2 (mod q).
4. The public key is (A, t). The secret key is (s1, s2).

### Signing

1. Sample a random masking vector y.
2. Compute w = A * y (mod q).
3. Compute a challenge c = Hash(message, w).
4. Compute z = y + c * s1.
5. Check that z is not too large (rejection sampling). If it is, restart.
6. The signature is (z, c).

### Verification

1. Compute w' = A * z - c * t (mod q).
2. Check that Hash(message, w') equals c.

The rejection sampling step is critical: it ensures that the signature z does not leak information about the secret key s1.

## Why Lattices Resist Quantum Attacks

Shor's algorithm works by exploiting the periodic structure of modular exponentiation (for RSA) or the group structure of elliptic curves (for ECC). Lattice problems do not have this kind of exploitable structure.

The best known quantum algorithms for lattice problems provide only a small polynomial speedup over classical algorithms — not the exponential speedup that Shor's gives for factoring. This is why lattice-based cryptography is considered quantum-resistant.

## Parameter Sizes Compared to Classical Crypto

The trade-off for quantum resistance is larger keys and ciphertexts:

| Algorithm | Public Key | Signature/Ciphertext | Security Basis |
|-----------|-----------|---------------------|----------------|
| RSA-2048 | 256 B | 256 B | Factoring (broken by Shor's) |
| ECDSA P-256 | 64 B | 64 B | ECDLP (broken by Shor's) |
| ML-KEM-768 | 1,184 B | 1,088 B | Module-LWE (quantum-resistant) |
| ML-DSA-65 | 1,952 B | 3,309 B | Module-LWE (quantum-resistant) |

The key sizes are larger but still practical for most applications. The performance overhead is modest — ML-KEM operations take single-digit milliseconds.
